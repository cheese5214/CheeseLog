# **CheeseLog Agent**

## **代码规范**

### **代码布局**

- 缩进

    4 个空格

- 空行

	根据功能块分割，每个功能块之间 1 个空行

- 注释

	对单行代码的注释使用 `#`，添加在该行代码后方，例如

	```python
	print('TEST') # xxxx
	```

	对代码块的注释使用 `` ''' ''' ``，如果只有一句话，则只写一行，如果有多句，则写多行，例如

	```python
	''' xxxx '''
	print('TEST')

	'''
	xxxx
	xxxx
	'''
	print('TEST')
	```

- 空格

	- 运算符、and、or

		前后都添加空格

	- 逗号、冒号

		前不加空格，后加空格

	- 英文、数字

		与中文混写的时候，前后都添加空格

- 推导式

	强制为三行，第一行和第三行为推导格式，第二行为生成逻辑

### **命名规范**

- 变量、函数

	使用下划线分割词语，全部小写

- 类名

	驼峰式

- 常量

	使用下划线分割词语，全部大写

- 私有变量、私有函数

	在变量、下划线的规范下，首位新增 `_`

- 协程

	如果变量、函数有同步方法，则异步首位新增 `async_`

### **导入规范**

除非需要动态导入，否则所有导入都应当在文件的最顶层

导入代码开分为三层，第一层为 python 内置库，第二层为第三方库，第三层为本地模块库；每块代码之间使用 1 行空格分隔

如果有多个变量从一个库中引入，只保留一行 `from` 入口，变量全部在 `import` 后一次导入

如果本地命名和导入命名有冲突，则导入命名的首字母大写，首位新增`Py`，作为别名

如果导入仅作为类型注释，则将其在 `from Typing import TYPE_CHECKING` 中导入

### **代码提示**

- 变量

	遵循代码布局-注释中的代码快注释

- 函数

	遵循 markdown 格式

	```python
	def xxx():
		'''
		介绍

		```python
		示例
		```

		- Args
			- xxx

				...

		- Kwargs
			- xxx

				...

		- Returns

			...

		- Exceptions

			...
		'''
	```

- 类型

	使用 ` | ` 语法，不使用 `Optional` 或者 `Union`

	使用内置泛型（例如 `dict[str, int]`），禁止使用 `typing.Dict`

### **代码结构**

- 引号

	统一使用单引号

- 类

	类必须使用 `__slots__`

- 字符串

	统一使用 f-string
